Task 1 — Lang + Resolve

Create internal/core/i18n/i18n.go · Test i18n_test.go

Precedencia CCP_LANG (env) → cfgLang → En; valor inválido cae a En. ResolveWithSource reporta la fuente para ccp lang.

func ResolveWithSource(cfgLang string) (Lang, Source) {
    if l, ok := normalize(os.Getenv("CCP_LANG")); ok { return l, SourceEnv }
    if l, ok := normalize(cfgLang); ok { return l, SourceConfig }
    return En, SourceDefault
}
// normalize: trim+lower, acepta solo en/es.

Task 2 — Motor T + register + completitud

Create catalog.go · Test catalog_test.go

T: falta lang → fallback En; falta key → devuelve la key cruda (ruidoso). register fusiona sub-catálogos por área y entra en pánico ante key duplicada. TestCatalogComplete exige en+es por key — la red de seguridad del barrido.

func T(l Lang, key string, a ...any) string {
    m, ok := catalog[key];      if !ok { return key }
    tmpl, ok := m[l];           if !ok { tmpl = m[En] }
    if len(a) == 0 { return tmpl }
    return fmt.Sprintf(tmpl, a...)
}

§ 03Traducción del CLI

Task 7 — barrido mecánico, guardado por TestCatalogComplete + parity + build.

Procedimiento por literal

  1. Key namespaced cli.<área>.<concepto>.
  2. Entrada {En:"…", Es:"<literal español actual byte-a-byte>"} en catalog_cli.go; traducir el EN.
  3. Resolver lang: con cfg → i18n.Resolve(cfg.Lang); sin cfg → currentLang().
  4. Reemplazar el literal por i18n.T(lang, "key", args…), moviendo %s/%d a la plantilla.
Constraint de paridad. La única prosa que el golden captura es el warning de _env perfil-faltante. Su ES debe calcar exacto (incluido el doble espacio tras ⚠️): ⚠️  ccp: perfil %s no existe; usando default. Como parity corre en CCP_LANG=es, se usa el ES y debe matchear byte-a-byte.

Archivos del barrido: cli.go, present.go (humanType→keys), status.go, config.go, doctor.go, install.go, instruct.go, help.go (el bloque grande), path.go, key.go, profile.go, backup.go, upgrade.go. Smoke: ccp help EN vs CCP_LANG=es ccp help ES; ninguna key cruda.
