§ 00Goal & Architecture

Goal: dar a Claude Code los comandos /ccp:remember-{global,profile,project}, /ccp:recall y /ccp:forget que persisten artefactos (rule/agent/command/hook/mcp/skill) en la estructura oficial de Claude Code, con CRUD seguro.

Architecture: un nuevo lib/instruct.sh (funciones puras: resolución de destino por scope×tipo, CRUD del bloque de reglas en CLAUDE.md, CRUD del manifest) y un cmd_instruct() en bin/ccp (add|list|rm|dest|record) lo exponen como CLI determinista y testeable. Cinco archivos markdown bajo commands/ccp/ son prompts que hacen que Claude clasifique la intención, redacte el contenido, confirme y llame al CLI. install.sh copia los comandos a ~/.claude/commands/ccp/, desde donde el symlink de cada cc-home los propaga a todos los perfiles.

Cara cliente: un verbo simple (remember / instructions). Cara Claude Code: estructura oficial nativa — ccp no inventa contenedor custom. (ADR 0005)

Convenciones obligatorias (CLAUDE.md)

  • BSD/macOS: nada de grep -P; filtrar con awk -F'\t'.
  • Pipes en tests: nunca pipear ccp a grep -q/head (SIGPIPE bajo pipefail); capturar y case-match.
  • Funciones puras: reciben ccp_home explícito; tests usan temp dirs.
  • Commits: solo con autorización explícita del turno; sin trailer Co-Authored-By.

§ 02Milestone 1 — Mecánica de reglas + destino

Entrega ccp instruct add|list|rm <scope> rule testeado + el resolvedor de destino completo. CLI puro, sin tocar config real.

Task 1.1 · ccp_instruct_dest (resolvedor scope×tipo)

Create: lib/instruct.sh · Test: tests/run.sh

Función pura que mapea (scope, type) → ruta, con códigos de retorno para los casos guarda: 2 profile+default, 3 tipo de archivo en profile, 4 project sin root, 1 desconocido.

# ccp_instruct_dest <scope> <type> <ccp_home> <profile> <src> <repo_root>
ccp_instruct_dest() {
  local scope="$1" type="$2" home="$3" prof="$4" src="$5" root="$6"
  local ov="$home/profiles/$prof/overlay"
  case "$scope" in
    global)  case "$type" in
      rule) printf '%s/CLAUDE.md' "$src" ;;
      hook|mcp) printf '%s/settings.json' "$src" ;;
      agent) printf '%s/agents' "$src" ;;
      command) printf '%s/commands' "$src" ;;
      skill) printf '%s/skills' "$src" ;;
      *) return 1 ;; esac ;;
    profile) [[ -n "$prof" && "$prof" != "default" ]] || return 2
      case "$type" in
      rule) printf '%s/CLAUDE.md' "$ov" ;;
      hook|mcp) printf '%s/settings.overlay.json' "$ov" ;;
      agent|command|skill) return 3 ;;
      *) return 1 ;; esac ;;
    project) [[ -n "$root" ]] || return 4
      case "$type" in
      rule) printf '%s/.claude/CLAUDE.md' "$root" ;;
      hook) printf '%s/.claude/settings.json' "$root" ;;
      mcp) printf '%s/.mcp.json' "$root" ;;
      agent) printf '%s/.claude/agents' "$root" ;;
      command) printf '%s/.claude/commands' "$root" ;;
      skill) printf '%s/.claude/skills' "$root" ;;
      *) return 1 ;; esac ;;
    *) return 1 ;;
  esac
}

Task 1.2 · CRUD del bloque de reglas

Modify: lib/instruct.sh · Test: tests/run.sh

Bloque con marcadores (idioma del shell-init del repo). add dedup (rc 9), list en orden, rm por índice 1-based.

CCP_INSTR_BEGIN='<!-- >>> ccp instructions >>> -->'
CCP_INSTR_END='<!-- <<< ccp instructions <<< -->'

ccp_instruct_rule_list() { # file
  local f="$1"; [[ -f "$f" ]] || return 0
  awk -v b="$CCP_INSTR_BEGIN" -v e="$CCP_INSTR_END" '
    $0==b{inb=1;next} $0==e{inb=0}
    inb && /^- /{line=$0; sub(/^- /,"",line); print line}' "$f"
}
ccp_instruct_rule_add() { # file text  (rc 9 = duplicado)
  local f="$1" text="$2"; ccp_instruct_block_ensure "$f"
  ccp_instruct_rule_list "$f" | grep -qxF "$text" && return 9
  local tmp; tmp="$(mktemp)"
  awk -v end="$CCP_INSTR_END" -v line="- $text" '$0==end{print line}{print}' "$f" > "$tmp" && mv "$tmp" "$f"
}

Task 1.3 · cmd_instruct en bin/ccp (rule)

Modify: bin/ccp (source lib · cmd_instruct · main() · help · completions) · Test: tests/run.sh

El binario resuelve perfil activo (CCP_PROFILE), src (CCP_CLAUDE_SRC) y raíz (_repo_root git → fallback $PWD). _instr_dest_or_die traduce los rc a mensajes en español.

_instr_repo_root() {
  [[ -n "${CCP_REPO_ROOT:-}" ]] && { printf '%s' "$CCP_REPO_ROOT"; return; }
  local r; r="$(_repo_root)"; [[ -n "$r" ]] && printf '%s' "$r" || printf '%s' "$PWD"
}
_instr_dest_or_die() { # scope type
  local dest rc; dest="$(ccp_instruct_dest "$1" "$2" "$CCP_HOME" \
    "$(_instr_active_profile)" "$CCP_CLAUDE_SRC" "$(_instr_repo_root)")"; rc=$?
  case "$rc" in
    0) printf '%s' "$dest" ;;
    2) err "perfil activo es 'default' (sin overlay)."; return 2 ;;
    3) err "tipo '$2' no existe a nivel perfil (usa global/project)."; return 3 ;;
    4) err "no estás en un repo git."; return 4 ;;
    *) err "scope/tipo inválido."; return 1 ;;
  esac
}

Dispatch: añadir instruct) cmd_instruct "$@" ;; en main(); instruct en las listas top de completions; nueva sección en cmd_help.

§ 04Milestone 3 — Artefactos de archivo + manifest

agent / command / skill como archivos en la estructura oficial, registrados en el manifest para CRUD seguro (nunca toca lo hecho a mano).

Task 3.1 · CRUD del manifest

Modify: lib/instruct.sh · Test: test_instruct_manifest_*

Fila: scope⇥profile⇥type⇥ref⇥desc. Split por localidad: $ccp_home/authored.tsv (global+profile, machine-local) vs $repo/.claude/ccp-authored.tsv (project, versionado).

ccp_instruct_manifest_file() { # scope ccp_home repo_root
  case "$1" in
    global|profile) printf '%s/authored.tsv' "$2" ;;
    project) printf '%s/.claude/ccp-authored.tsv' "$3" ;;
    *) return 1 ;;
  esac
}
ccp_instruct_manifest_rm() { # manifest scope profile index → imprime "type⇥ref"
  local m="$1" scope="$2" prof="$3" idx="$4"; [[ -f "$m" ]] || return 1
  local target; target="$(awk -F'\t' -v s="$scope" -v p="$prof" -v want="$idx" \
    '$1==s && (s!="profile"||$2==p){c++; if(c==want){print NR; exit}}' "$m")"
  [[ -n "$target" ]] || return 1
  sed -n "${target}p" "$m" | cut -f3,4
  local tmp; tmp="$(mktemp)"; awk -v t="$target" 'NR!=t' "$m" > "$tmp" && mv "$tmp" "$m"
}

ccp_instruct_json_merge() { # file snippet_json (requiere jq)
  local f="$1" snippet="$2"; command -v jq >/dev/null 2>&1 || return 1
  [[ -f "$f" ]] && jq -e . "$f" >/dev/null 2>&1 || printf '{}\n' > "$f"
  printf '%s' "$snippet" | jq -e . >/dev/null 2>&1 || return 1
  local tmp; tmp="$(mktemp)"
  jq --argjson add "$snippet" '. * $add' "$f" > "$tmp" && mv "$tmp" "$f"
}
