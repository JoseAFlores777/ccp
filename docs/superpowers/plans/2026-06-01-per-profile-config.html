§ 01Goal & arquitectura

Qué construimos y la forma general del enfoque.

Goal. Permitir que cada perfil cargue su propia config de Claude Code (instrucciones + hooks/permisos/env/MCP) como capa baseline, editable desde la CLI con un editor configurable.

Arquitectura. Todo perfil no-default tiene un cc-home (= CLAUDE_CONFIG_DIR). El perfil guarda dos archivos overlay: overlay/CLAUDE.md y overlay/settings.overlay.json. El cc-home/CLAUDE.md efectivo hace @import del global y del overlay; el cc-home/settings.json efectivo es global ⊕ overlay vía jq deep-merge (con fallback snapshot si falta jq). La regeneración corre al crear/editar/sync — nunca en el hook por-prompt.

Commits no autónomos. Cada paso Commit es una propuesta: stagea, muestra el diff y pregunta antes de git commit. Sin trailer Co-Authored-By.

§ 04Dataflow del merge

Cómo el global y el overlay producen el cc-home efectivo.

La regeneración se dispara solo en create/edit/sync. El hook por-prompt (_ccp_autocheck) nunca corre jq: solo evalúa el delta de env. El overlay md se lee vivo vía @import — editarlo no requiere regen.

Task 2 — Validar JSON + merge de settings

Modify lib/cfg.sh

ccp_cfg_validate_json (no-op sin jq) y ccp_cfg_merge_settings: deep-merge recursivo de objetos (overlay gana), arrays se reemplazan; sin jq cae a snapshot.

ccp_cfg_merge_settings() { # global overlay out
  local g="$1" o="$2" out="$3"
  if command -v jq >/dev/null 2>&1; then
    local -a files=()
    [[ -f "$g" ]] && jq -e . "$g" >/dev/null 2>&1 && files+=("$g")
    files+=("$o")
    jq -s 'reduce .[] as $x ({}; . * $x)' "${files[@]}" > "$out"
  else
    [[ -f "$g" ]] && cp "$g" "$out" || cp "$o" "$out"
  fi
}

Task 3 — Generar cc-home/CLAUDE.md (@import) + regenerate

Modify lib/cfg.sh

Trampa crítica. Si cc-home/CLAUDE.md es un symlink viejo, escribir con > corromper­ía el global apuntado. ccp_cfg_write_claude_md hace rm -f del symlink antes de escribir. Test dedicado verifica que el global no se pisa.

ccp_cfg_write_claude_md() { # home name src
  ...
  [[ -L "$cch/CLAUDE.md" ]] && rm -f "$cch/CLAUDE.md"
  {
    printf '# %s — generado por ccp ...\n\n' "$name" "$name"
    [[ -f "$src/CLAUDE.md" ]] && printf '@%s\n' "$src/CLAUDE.md"
    printf '@%s\n' "$overlay"
  } > "$cch/CLAUDE.md"
}

Task 4 — Migración legacy → overlay

Modify lib/cfg.sh

ccp_cfg_migrate_legacy: si cc-home/settings.json es copia real y no hay overlay aún → lo mv al overlay (preserva ediciones). Symlink viejo de CLAUDE.md → rm. Idempotente: no pisa un overlay ya migrado.

§ 06Task 5 · lib/env.sh

deepseek pasa de env-only a tener su propio config dir.

En la rama deepseek), primer printf:

printf 'export CLAUDE_CONFIG_DIR=%q\n' "$home/profiles/$profile/cc-home"

Esto rompe a propósito test_env_deepseek (antes esperaba CLAUDE_CONFIG_DIR sin setear). El test se actualiza para esperar el cc-home. Ver ADR-0003.

Menú TTY (1 instrucciones / 2 settings / 3 ambos). Target directo (instructions|settings) salta el menú para scripting; sin TTY y sin target → error. Settings se valida con jq: JSON malo ⇒ warn y no regenera (conserva el último bueno). default abre ~/.claude real con aviso.

Task 9 — ccp profile sync

Re-mergea global ⊕ overlay para un perfil (o todos). Resuelve el staleness cuando el global cambia tras crear el perfil.
