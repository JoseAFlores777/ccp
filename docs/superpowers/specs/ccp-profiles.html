§ 01Overview

De un booleano a un nombre de perfil.

Hoy dsctl alterna Claude Code entre tu login oficial de Anthropic y DeepSeek, con scope por terminal y por carpeta vía un hook precmd. El modelo es binario: un path resuelve a deepseek u official.

Este spec lo generaliza a perfiles con nombre. Una carpeta resuelve a un nombre de perfil, donde un perfil es:

  • una cuenta oficial de Anthropic (con su propio CLAUDE_CONFIG_DIR),
  • un provider compatible tipo DeepSeek (su base URL, key y modelos), o
  • el perfil reservado default (tu login normal en ~/.claude).

Repo A → cuenta work; repo B → cuenta personal; repo C → deepseek. El binario y la función shell se renombran a ccp (un solo nombre para ambos).

§ 02Goals / Non-goals

Goals

  • Enrutar cada path a un perfil arbitrario (≥2 cuentas oficiales + ≥1 provider).
  • Mantener scope por-terminal/por-carpeta y el auto-switch en cd.
  • Single source of truth del env: el binario emite el delta, el shell lo aplica.
  • Auto-migrar ~/.config/dsctl; on/off legacy siguen vivos.
  • Preservar la superficie machine-readable (resolve, status --json) con awareness de perfil.

Non-goals

  • Sin tipo ANTHROPIC_API_KEY (Console) en v1 — solo OAuth + provider.
  • No automatizar el OAuth: ccp enruta env y lanza claude; tú corres /login una vez por perfil oficial.
  • Sin árboles de reglas con herencia por capas (una regla más-específica gana).

§ 03Conceptos: perfiles

Un perfil = nombre + tipo + datos del tipo.

TipoDatosActivación (delta de env)
officialdir cc-home/ propioexport CLAUDE_CONFIG_DIR + unset todo ANTHROPIC_*
deepseek (provider)base_url, api_key (600), model_pro, model_flash, effortexport ANTHROPIC_* + unset CLAUDE_CONFIG_DIR
default (reservado)—unset todo lo manejado (usa ~/.claude)

default siempre existe, es el fallback cuando ninguna regla aplica, y es asignable a un path para abrir un hueco oficial dentro de un subárbol provider.

Resolución

Para un path P: entre las reglas cuyo path es P o ancestro de P, gana la más específica (más profunda). Los paths son únicos → no hay empates por profundidad. Sin regla → default. (La vieja regla "exclude gana empate exacto" desaparece: ya no existe exclude.)

§ 04Motor de reglas de path

Las líneas de rules.tsv pasan a path<TAB>profile. El resolver de funciones puras (hoy lib/paths.sh) mantiene most-specific-wins pero devuelve un nombre de perfil en vez de deepseek|official. Sin match → default.

# asigna ruta → perfil (reemplaza include/exclude)
ccp path set ~/work        work
ccp path set ~/work/cliente default      # carve-out
ccp path set ~/lab         deepseek
ccp path rm   <ruta>
ccp path list                            # reglas + regla efectiva del cwd
ccp path test <ruta>                      # imprime el perfil (scriptable)
ccp path clear | edit

§ 05Aplicación de env

El binario corre en proceso hijo: no puede mutar el shell padre.

  • Comando interno ccp _env <perfil> imprime un delta completo eval-able: unset de cada var manejada, luego export del perfil objetivo. Stateless — aplicar el delta de cualquier perfil desde cualquier estado previo es correcto.
  • La función ccp y el hook hacen eval "$(ccp _env <perfil>)".
  • Optimización: ccp _hook "$PWD" resuelve e imprime el delta en un solo fork, preservando el cache "fork solo si cambió $PWD".

Vars manejadas: CLAUDE_CONFIG_DIR, ANTHROPIC_BASE_URL, ANTHROPIC_AUTH_TOKEN, ANTHROPIC_MODEL, ANTHROPIC_DEFAULT_{OPUS,SONNET,HAIKU}_MODEL, CLAUDE_CODE_SUBAGENT_MODEL, CLAUDE_CODE_EFFORT_LEVEL, + marcador CCP_PROFILE.

Seguridad. El hook hace eval de la salida del binario. Esa salida se genera de tu propia config, va fuertemente quoteada, y el binario es confiable. Ningún input externo llega a _env.

§ 06Config dir oficial & herencia

CLAUDE_CONFIG_DIR reubica todo ~/.claude (plugins, settings, MCP, skills, history). Un perfil oficial nuevo arrancaría pelado. Por eso al hacer ccp profile add <name> --official, se siembra cc-home/ con symlinks selectivos desde ~/.claude.

Compartido (symlink → ~/.claude)Separado (por perfil)
plugins/, commands/, agents/, skills, CLAUDE.mdcredenciales/keychain, history.jsonl, projects/, .claude.json
settings.json es el delicado. Claude Code escribe ahí (approvals, "use custom API key"). Default: copiar (no symlink) para que los writes por-cuenta no se filtren. (Pendiente 12.1 — confirmar.)

§ 07Layout de almacenamiento

  profiles.tsv            # name<TAB>type  (índice)
  profiles/
    <name>/
      meta                # type + fields (base_url, models, effort)
      api_key             # 600, solo provider

§ 08Superficie CLI

ccp                          # menú interactivo
ccp use <perfil>             # (shell) activa perfil en esta terminal
ccp default | off            # (shell) vuelve a ~/.claude
ccp run [cmd]                # (shell) corre cmd/claude con el perfil del cwd
ccp on                       # (shell, legacy) alias → use deepseek

ccp profile add <name> --official | --deepseek
ccp profile rm <name>
ccp profile list | show <name>
ccp profile login <name>     # lanza claude con el cc-home del perfil para /login

ccp path set|rm|list|test|clear|edit
ccp resolve [ruta]           # nombre de perfil (exit 0=no-default,1=default)
ccp status [--json]          # --json gana 'profile' + 'profile_type'
ccp install|uninstall|key|config|completion|doctor|version|help

account es alias de profile (nice-to-have).

§ 09Función shell + hook

La función ccp() intercepta use/default/off/run/on (mutación de env) y cae a command ccp "$@" para lo demás — espejo del ds() actual. El hook precmd (_ccp_autocheck) cachea por $PWD, llama ccp _hook "$PWD" y hace eval del delta. El completion auto-carga.

Caveat de sesión viva. Claude Code lee el env al arrancar. Cambiar el perfil (por cd o ccp use) afecta el próximo claude, no una sesión ya corriendo — igual que hoy con DeepSeek. Se documenta, no se arregla.

§ 10Migración dsctl → ccp

  1. Si existe ~/.config/dsctl y no ~/.config/ccp: crear perfil builtin deepseek desde el config viejo (base_url/models/effort) + api_key.
  2. Convertir reglas: include→asignar deepseek, exclude→asignar default.
  3. Backup del rules.tsv viejo.
  4. ccp install detecta el bloque # >>> dsctl shell init >>> en el rc y ofrece quitarlo.

Verbos legacy: ccp on→use deepseek, ccp off→default.

§ 11Superficie scripting

  • ccp resolve [ruta] → imprime nombre de perfil; exit 0 = perfil no-default activo, 1 = default. (El caso deepseek preserva el viejo 0.)
  • ccp status --json → campos actuales + "profile", "profile_type".

§ 13Criterios de aceptación

  • ccp profile add work --official + ccp profile login work → logueado en 2da cuenta sin perturbar default.
  • ccp path set ~/repoA work && cd ~/repoA && claude → arranca como work; cd ~/repoC (deepseek) → DeepSeek; cd ~ → default.
  • Config dsctl existente auto-migra; bloque rc ds/dsctl quitable; ccp on/off siguen.
  • ccp resolve / status --json profile-aware; smoke test de CI del motor actualizado a nombres de perfil.
  • ccp help documenta toda la superficie en español.
