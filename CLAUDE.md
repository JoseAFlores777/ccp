# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`dsctl` is a Bash CLI that switches Claude Code between the user's official Anthropic login and DeepSeek models. The switch is scoped **per terminal** and **per directory** — never global — so DeepSeek's `ANTHROPIC_BASE_URL`/`ANTHROPIC_AUTH_TOKEN` env vars don't leak into company/official sessions. User-facing strings are in Spanish.

## Commands

```bash
shellcheck -S warning bin/dsctl lib/paths.sh install.sh   # lint (CI gate, non-blocking)
bash -n bin/dsctl && bash -n lib/paths.sh                 # syntax check
```

Run the path-engine smoke test (same as CI, `.github/workflows/ci.yml`):

```bash
source lib/paths.sh
RF=$(mktemp)
ds_rule_add include /a "$RF"
ds_rule_add exclude /a/b "$RF"
ds_rule_add include /a/b/c "$RF"
test "$(ds_resolve /a/x "$RF")" = deepseek      # inherits include
test "$(ds_resolve /a/b/y "$RF")" = official    # exclude wins
test "$(ds_resolve /a/b/c/z "$RF")" = deepseek  # deeper include re-enables
```

`lib/paths.sh` is pure functions taking an explicit rules-file arg — test it directly without touching real config in `~/.config/dsctl`.

## Architecture

The binary/shell-function split is the central design constraint:

- **`bin/dsctl`** — the CLI binary. Owns rules, key, config, menu, path resolution. Runs in a child process, so it **cannot** mutate the parent shell's environment.
- **Shell function `ds`** — generated by `_print_shell_init()` in `bin/dsctl` and appended to the user's `~/.zshrc`/`~/.bashrc` by `dsctl install`. Because it runs *in* the shell, only it can `export`/`unset` the `ANTHROPIC_*` vars. `ds on/off/run` does the env toggling; anything else falls through to `command dsctl "$@"`.
- **`_ds_autocheck` hook** — also generated into the rc; runs on every prompt (zsh `precmd_functions` / bash `PROMPT_COMMAND`). Calls `dsctl _resolve "$PWD"` and turns `ds on`/`off` automatically based on the verdict. This is why `cd` alone flips the mode.

So: env mutation lives in the rc-installed function; all logic lives in the binary; the hook bridges them by polling `_resolve` per directory.

### Path rule engine (`lib/paths.sh`)

Resolution for a path P: among all `include`/`exclude` rules that are P or an ancestor of P, the **most specific** (deepest path) wins. Exact-depth tie → `exclude` wins. No matching rule → `official`. Rules stored as `kind<TAB>/abs/path` lines in `~/.config/dsctl/rules.tsv`. `ds_norm_path` resolves `.`/`..`/`~` textually (not `realpath`) so non-existent paths still work.

### Config & state locations

- `~/.config/dsctl/api_key` — DeepSeek key, `chmod 600`, never in rc or git.
- `~/.config/dsctl/rules.tsv` — path rules.
- `~/.config/dsctl/config` — sourced shell file with `DS_BASE_URL`, `DS_MODEL_PRO`, `DS_MODEL_FLASH`, `DS_EFFORT`. Defaults live as `*_DEFAULT` vars near the top of `bin/dsctl`; the generated `ds` function hardcodes the same defaults independently, so changing a default means editing **both** places.

`install.sh` copies `bin/dsctl` → `~/.local/bin` and `lib/paths.sh` → `~/.local/lib/dsctl`; the binary finds its lib next to itself or falls back to that installed path (`DSCTL_LIB` logic at the top of `bin/dsctl`).

## Conventions

- Commands dispatch through `main()` → `cmd_*` functions; the `_resolve` command is internal (used only by the hook).
- Output helpers `ok/warn/err/info/say/hr` respect `NO_COLOR` and non-TTY. Keep user-facing text Spanish.
- When editing the `ds` function or hook, edit the heredoc inside `_print_shell_init()` — that text *is* what gets installed. `cmd_uninstall` strips it by matching the `# >>> dsctl shell init >>>` / `# <<<` markers, so don't change those markers casually. The heredoc also auto-loads completion (`eval "$(dsctl completion zsh|bash)"`) and the `_ds_autocheck` hook caches by `$PWD` to avoid forking `dsctl` on every prompt.
- **macOS portability**: the binary runs under `bash` but coreutils are BSD. Do NOT use `grep -P` (BSD grep has no `-P`) — filter with `awk -F'\t'` or a `while IFS=$'\t' read` loop instead. This bit `path list` once (it silently showed no rules on macOS). Test new shell code with `/usr/bin/grep`/`/usr/bin/sort`, not the GNU tools that may shadow them in an interactive shell.
- Machine-readable surface for scripting: `dsctl resolve [path]` and `dsctl path test [path]` set exit codes (0=deepseek, 1=official); `dsctl status --json` emits a JSON object. Keep these stable.
