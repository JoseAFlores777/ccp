Claude Code Profiles_

Una cuenta de Claude Code distinta en cada carpeta. En tu repo de trabajo, tu cuenta de empresa; en tu proyecto personal, tu cuenta personal; en tus experimentos, DeepSeek. El cambio ocurre solo, con hacer cd.

// oficial

Cuenta Anthropic

Tu login de empresa o personal, cada uno aislado.

// proveedor

DeepSeek & cía

Un endpoint compatible con su propia API key.

// default

Tu login normal

Tu ~/.claude de siempre. Sin reglas, gana este.

macOS · Linux bash · zsh requiere Claude Code + git
←→ navega entre páginas · G abre el índice
§ 00Empieza aquí

El modelo mental — 30 segundos

  1. Un perfil es una identidad: una cuenta oficial de Anthropic, un proveedor tipo DeepSeek, o tu login normal (default).
  2. Una regla dice "esta carpeta (y sus subcarpetas) usa tal perfil".
  3. Gana la regla más específica. Sin regla → tu login normal.

Cuando entras a una carpeta, ccp aplica el perfil correcto en esa terminal mediante un hook. Luego corres claude normal.

Pruébalo · ¿qué perfil gana?
cd

resuelve a

¿por qué "la más profunda"?

Las reglas anidan. ~/work cubre todo lo de adentro, pero si una subcarpeta tiene su propia regla, esa es más específica y manda solo ahí. Así puedes tener una excepción puntual sin tocar la regla del padre. Eso es el carve-out del Caso 04.

·Una sola vez

Instalación

Cinco pasos. Cópialos en orden.

01

Instala el binario y las librerías.

./install.sh
02

~/.local/bin en tu PATH. Solo si el paso 1 te advirtió, añade a ~/.zshrc o ~/.bashrc:

export PATH="$HOME/.local/bin:$PATH"
¿qué es el PATH?

La lista de carpetas donde tu shell busca comandos. Si ccp no está ahí, el shell no lo encuentra y verás command not found.

03

Instala la función de shell y el hook automático.

ccp install
¿por qué hace falta una función?

El binario corre en un proceso hijo y no puede cambiar el entorno de tu terminal. La función de shell sí: por eso ccp install la agrega a tu rc, y ella aplica el perfil con export.

04

Recarga tu shell — usa el archivo de TU shell.

source ~/.zshrc     # zsh
source ~/.bashrc    # bash
05

Confirma que quedó bien.

ccp doctor

Si ves "Función 'ccp' cargada", estás listo.

§ 01El más común

Tu cuenta de trabajo en tu carpeta de trabajo

El flujo base. Los demás casos son variaciones de este.

01

Crea el perfil oficial "work".

ccp profile add work --official
02

Inicia sesión en ese perfil (una sola vez).

ccp profile login work

Dentro de Claude Code: escribe /login, autentícate con tu cuenta de trabajo, y sal con /quit.

03

Dile a ccp qué carpeta usa "work".

ccp path set ~/work work
04

Pruébalo: entra y abre Claude.

cd ~/work && claude

Arranca con tu cuenta de trabajo. No tuviste que hacer nada más.

§ 02Otra identidad

Añadir tu cuenta personal

Igual que el caso 1, con otro nombre y otra carpeta.

01

Crea el perfil.

ccp profile add personal --official
02

Inicia sesión (una vez), con tu cuenta personal.

ccp profile login personal

Dentro: /login con tu cuenta personal, luego /quit.

03

Asigna tu carpeta personal.

ccp path set ~/personal personal

~/work sigue siendo trabajo; ~/personal ahora es personal.

§ 03Proveedor

Una carpeta con DeepSeek

Para experimentos o trabajo donde prefieras DeepSeek en vez de tu cuenta Anthropic.

01

Crea el perfil de proveedor.

ccp profile add deepseek --deepseek
02

Guarda tu API key de DeepSeek (te la pide oculta).

ccp key deepseek
¿dónde queda la key?

En ~/.config/ccp/profiles/deepseek/api_key con permisos 600. Nunca va al YAML, ni al rc, ni a git.

03

Asigna la carpeta.

ccp path set ~/labs deepseek
04

Úsalo.

cd ~/labs && claude
§ 04Excepción

Sacar una subcarpeta de su regla (carve-out)

Tu ~/work usa la cuenta de trabajo, pero hay un cliente puntual donde quieres tu login normal.

Un solo paso — asigna esa subcarpeta al perfil default.

ccp path set ~/work/cliente-x default
Por qué funciona~/work sigue en "work", pero ~/work/cliente-x usa tu login normal. La subcarpeta más específica siempre manda. ¿Quieres verlo? Vuelve al playground del modelo mental y escribe ~/work/cliente-x.
§ 05Manual

Cambiar a mano en una terminal

Cuando quieres cambiar el perfil de la terminal actual sin moverte de carpeta.

Activar un perfil aquí:

ccp use personal

Volver a tu login normal:

ccp default

Correr Claude una sola vez con el perfil de la carpeta, sin dejarlo fijo:

ccp run claude
§ 06Inspección

Ver qué está pasando

¿Qué perfil usa esta carpeta y qué hay activo en la terminal?

ccp status

¿Qué reglas de carpeta tengo?

ccp path list

¿Qué perfiles tengo creados?

ccp profile list

¿Está todo bien (logins, keys, función de shell)?

ccp doctor
§ 07Migración

Vengo de dsctl (la versión vieja)

ccp reemplaza a dsctl. La migración es automática.

01

La primera vez que corras ccp, migra solo: crea un perfil deepseek con tu config vieja y convierte tus reglas. Solo úsalo:

ccp status
02

Quita el bloque viejo de dsctl de tu shell.

dsctl uninstall

Tus reglas viejas quedan respaldadas en ~/.config/ccp/rules.dsctl.bak.

·Avanzado · Versiones

Actualizar ccp

De una versión anterior a la última. Tu config se migra sola; no pierdes perfiles ni reglas.

¿Qué versión tengo?

Imprime la versión del binario.

ccp version     # → 2.5.1

También vale ccp --version o ccp -v.

Subir a la última

Un comando. Reinstala el binario y re-sincroniza tus perfiles.

ccp upgrade

Descarga el binario de release para tu OS/arch, verifica su sha256, y luego regenera cada cc-home. Si no hay release pero sí toolchain de Go, compila desde fuente.

¿qué hace por dentro?

1. Lee ~/.config/ccp/install-source (la ruta del repo, registrada al instalar).

2. Re-ejecuta install.sh desde ahí: detecta OS/arch, baja el binario, verifica checksum, y borra las libs viejas de bash (~/.local/lib/ccp).

3. Corre ccp profile sync para regenerar la config de cada perfil.

Si el bloque de shell quedó viejo, te avisa: corre ccp install y recarga (source ~/.zshrc).

Variantes.

ccp upgrade --pull       # git pull en el repo antes de reinstalar
ccp upgrade --no-sync    # solo el binario, no toca los perfiles

Re-sincronizar sin reinstalar

Regenera CLAUDE.md + settings.json de cada perfil, re-fundiendo global ⊕ overlay. Útil tras cambiar tu config global o el overlay de un perfil.

ccp profile sync          # todos
ccp profile sync work     # solo uno

No toca default (no tiene overlay ni cc-home).

Vienes de la versión legacy (Bash / dsctl)

La migración es automática y perezosa: la primera vez que corras cualquier comando de ccp que toque la config, convierte tu estado viejo a ccp.yaml (schema v2). No tienes que hacer nada especial.

Qué pasa exactamente Antes de tocar nada, ccp respalda todo en ~/.config/ccp/.backup-pre-go-<fecha>. Crea un perfil deepseek con tu config DeepSeek vieja (la API key se mueve a api_key, chmod 600) y convierte tus reglas: las de include → deepseek, las de exclude → default. Es idempotente: corre cuantas veces quieras, migra una sola vez.

Después, quita el bloque viejo de tu shell — está en el Caso 07 · Vengo de dsctl.

·Avanzado · Respaldo

Exportar e importar tu configuración

Llévate todo a otra máquina, o respáldalo antes de un cambio grande. Todo cabe en un .tar.gz.

Exportar

Crear un respaldo (sin secretos).

ccp backup export ~/ccp-backup.tar.gz

Incluye manifest.yaml, tu ccp.yaml y los overlays de cada perfil. El archivo queda chmod 644.

Incluir secretos (API keys + logins oficiales).

ccp backup export ~/ccp-backup.tar.gz --with-secrets
CuidadoAñade tu api_key de DeepSeek y las credenciales OAuth de los perfiles oficiales (.claude.json). El archivo se escribe chmod 600. Guárdalo en un lugar seguro y nunca lo subas a git.
¿qué entra y qué no?

Siempre: manifest.yaml (con checksums sha256), ccp.yaml, y overlay/ de cada perfil.

Solo con --with-secrets: profiles/<n>/api_key y cc-home/.claude.json.

Nunca: los symlinks (plugins/ commands/ agents/ skills/). Se vuelven a crear solos al restaurar.

Importar / restaurar

Restaurar en otra máquina (o sobre esta).

ccp backup restore ~/ccp-backup.tar.gz

Por defecto: no pisa nada. Crea los perfiles que falten y fusiona las reglas (rutas nuevas se agregan, las tuyas se conservan).

Modos de colisión — qué pasa con lo que ya tienes:

ComandoPerfiles existentesReglas
restorese conservan; solo agrega los nuevosfusiona
restore --overwritereemplaza los que vengan en el backupfusiona
restore --forceborra todo y restaura limpioreemplaza todo
Red de seguridadAntes de tocar nada, ccp guarda un snapshot automático en ~/.config/ccp/.backup-pre-restore-<fecha>. Si algo sale mal, ahí está tu estado anterior. Y si el backup viene de un ccp más nuevo (schema superior), el restore aborta sin tocar nada: actualiza ccp primero.
·Avanzado · YAML

Configurar a mano (ccp.yaml)

Todo vive en un solo archivo. Puedes tocarlo con comandos o editarlo a mano: ambos caminos llevan al mismo YAML.

Con comandos

Ver los defaults actuales.

ccp config show

Cambiar un campo de los defaults. Claves: base_url, model_pro, model_flash, effort.

ccp config set model_pro deepseek-reasoner

Solo afecta a los perfiles deepseek que crees después. Los existentes guardan sus 4 campos explícitos.

Volver los defaults a fábrica · elegir editor.

ccp config reset
ccp config editor nano     # o vim, code, etc.

El archivo, anotado

Vive en ~/.config/ccp/ccp.yaml (o $CCP_HOME/ccp.yaml).

oficial vs deepseek

Un perfil oficial guarda solo type: official; su login vive aparte, en cc-home/.claude.json.

Un perfil deepseek guarda los 4 campos (base_url, model_pro, model_flash, effort) explícitos, sin herencia. defaults solo los siembra al crearlo.

Reglas de oro 1. default es implícito: nunca lo pongas en profiles. Es tu login normal y gana cuando ninguna regla aplica. Para una excepción, usa profile: default en una regla.
2. Las rutas en rules son absolutas y normalizadas (sin ~). Mejor deja que ccp las normalice con ccp path set ~/x perfil.
3. La API key no va aquí: vive en ~/.config/ccp/profiles/<n>/api_key (chmod 600). Edítala con ccp key <n>.
¿es seguro editarlo a mano?

Sí. ccp escribe de forma atómica (tmp + rename) bajo un flock, conserva tus comentarios y cualquier clave desconocida en round-trip.

Guard de versión: si version es mayor que la que tu binario conoce, ccp se niega a leer o escribir y no toca el archivo. Actualiza ccp.

Tras editar a mano: refresca el hook con cd . y verifica con ccp status · ccp path list.

·Avanzado · Idioma

Idioma

ccp habla inglés por defecto y también español. Elige el que prefieras; la elección se persiste en ccp.yaml.

Ver el idioma actual y de dónde sale.

ccp lang        # idioma actual + origen (env/config/default)

Cambiar de idioma y persistirlo en ccp.yaml.

ccp lang en     # inglés
ccp lang es     # español
Prioridad y atajos 1. CCP_LANG=en|es es un override por entorno y tiene prioridad sobre la config. El valor por defecto es inglés.
2. En el dashboard interactivo (TUI), pulsa L para cambiar el idioma en vivo — se persiste. Ojo: iniciar sesión (login) está en la l minúscula, y L (mayúscula) cambia el idioma.
·Ayuda

Solución de problemas

Toca una pregunta para ver la respuesta.

+ccp: command not found

~/.local/bin no está en tu PATH. Vuelve al Paso 2 de Instalación.

+ccp use … no cambia nada / "Función 'ccp' no cargada"

Falta cargar la función de shell. Corre ccp install y luego source ~/.zshrc (o ~/.bashrc).

+Cambié de carpeta y el perfil no cambió

El hook recuerda la última carpeta. Tras editar reglas, refresca con cd .

+Al abrir Claude dice "Not logged in"

Ese perfil oficial todavía no tiene sesión. Inícialo una vez con ccp profile login <nombre>.

+Quiero ver el perfil de una carpeta sin entrar en ella

Usa ccp resolve ~/ruta/que/sea
