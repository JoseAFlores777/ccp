Guía de uso

ccp

Una cuenta de Claude Code distinta en cada carpeta. En tu repo de trabajo, tu cuenta de empresa; en tu proyecto personal, tu cuenta personal; en tus experimentos, DeepSeek. El cambio ocurre solo, con hacer cd.

Empieza aquíEl modelo mental — 30 segundos

Tres ideas y ya:

1. Un perfil es una identidad: una cuenta oficial de Anthropic, un proveedor tipo DeepSeek, o tu login normal (default).
2. Una regla dice "esta carpeta (y sus subcarpetas) usa tal perfil".
3. Gana la regla más específica. Sin regla → tu login normal.

Cuando entras a una carpeta, ccp aplica el perfil correcto en esa terminal. Luego corres claude normal.

RequisitosAntes de empezar

Necesitas: macOS o Linux con bash o zsh, Claude Code instalado (que claude --version funcione), y git.

Una sola vezInstalación

Un paso a la vez.

1

Instala el binario y las librerías.

./install.sh
2

Asegúrate de que ~/.local/bin está en tu PATH. Solo si el paso 1 te mostró una advertencia, añade esto a tu ~/.zshrc o ~/.bashrc:

export PATH="$HOME/.local/bin:$PATH"
3

Instala la función de shell y el hook automático.

ccp install
4

Recarga tu shell — usa el archivo de TU shell.

source ~/.zshrc     # zsh
source ~/.bashrc    # bash
5

Confirma que quedó bien.

ccp doctor

Si ves "Función 'ccp' cargada ✅", estás listo.

Caso 1 · el más comúnTu cuenta de trabajo en tu carpeta de trabajo

1

Crea el perfil oficial "work".

ccp profile add work --official
2

Inicia sesión en ese perfil (una sola vez). Abre Claude Code con la config aislada de "work".

ccp profile login work

Dentro de Claude Code: escribe /login, autentícate con tu cuenta de trabajo, y sal con /quit.

3

Dile a ccp qué carpeta usa "work".

ccp path set ~/work work
4

Pruébalo: entra y abre Claude.

cd ~/work && claude

Arranca con tu cuenta de trabajo. No tuviste que hacer nada más.

Caso 2Añadir tu cuenta personal

Igual que el caso 1, con otro nombre y otra carpeta.

1

Crea el perfil.

ccp profile add personal --official
2

Inicia sesión (una vez), con tu cuenta personal.

ccp profile login personal

Dentro: /login con tu cuenta personal, luego /quit.

3

Asigna tu carpeta personal.

ccp path set ~/personal personal

~/work sigue siendo trabajo; ~/personal ahora es personal.

Caso 3Una carpeta con DeepSeek

Para experimentos o trabajo donde prefieras DeepSeek en vez de tu cuenta Anthropic.

1

Crea el perfil de proveedor.

ccp profile add deepseek --deepseek
2

Guarda tu API key de DeepSeek (te la pide oculta).

ccp key deepseek
3

Asigna la carpeta.

ccp path set ~/labs deepseek
4

Úsalo.

cd ~/labs && claude

Caso 4Sacar una subcarpeta de su regla (carve-out)

Tu ~/work usa la cuenta de trabajo, pero hay un cliente puntual donde quieres tu login normal.

Un solo paso — asigna esa subcarpeta al perfil default.

ccp path set ~/work/cliente-x default

~/work sigue en "work", pero ~/work/cliente-x usa tu login normal. La subcarpeta más específica siempre manda.

Caso 5Cambiar a mano en una terminal

Cuando quieres cambiar el perfil de la terminal actual sin moverte de carpeta.

Activar un perfil aquí:

ccp use personal

Volver a tu login normal:

ccp default

Correr Claude una sola vez con el perfil de la carpeta, sin dejarlo fijo:

ccp run claude

Caso 6Ver qué está pasando

¿Qué perfil usa esta carpeta y qué hay activo en la terminal?

ccp status

¿Qué reglas de carpeta tengo?

ccp path list

¿Qué perfiles tengo creados?

ccp profile list

¿Está todo bien (logins, keys, función de shell)?

ccp doctor

Caso 7Vengo de dsctl (la versión vieja)

ccp reemplaza a dsctl. La migración es automática.

1

La primera vez que corras ccp, migra solo: crea un perfil deepseek con tu config vieja y convierte tus reglas. Solo úsalo:

ccp status
2

Quita el bloque viejo de dsctl de tu shell.

dsctl uninstall

Tus reglas viejas quedan respaldadas en ~/.config/ccp/rules.dsctl.bak.

AyudaSolución de problemas

ccp: command not found

~/.local/bin no está en tu PATH. Vuelve al Paso 2 de Instalación.

ccp use ... no cambia nada / "Función 'ccp' no cargada"

Falta cargar la función de shell. Corre ccp install y luego source ~/.zshrc (o ~/.bashrc).

Cambié de carpeta y el perfil no cambió

El hook recuerda la última carpeta. Tras editar reglas, refresca con cd .

Al abrir Claude dice "Not logged in"

Ese perfil oficial todavía no tiene sesión. Inícialo una vez con ccp profile login <nombre>.

Quiero ver el perfil de una carpeta sin entrar en ella

ccp resolve ~/ruta/que/sea
